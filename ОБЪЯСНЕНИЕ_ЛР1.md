# Лабораторна робота №1 - Вступ до веб-розробки

## Опис роботи
Лабораторна робота №1 є вступним завданням з основ веб-розробки. Вона демонструє створення простого веб-сайту з використанням базових технологій: HTML для структури, CSS для стилізації та JavaScript для інтерактивності.

---

## Структура проекту

### **Файли проекту:**
- `index.html` - основна HTML сторінка
- `style.css` - файл стилів CSS  
- `script.js` - файл JavaScript

---

## Аналіз HTML структури (index.html)

```html
<!DOCTYPE html>
<html>
<head>
    <title>My First Lab Website</title>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <h1>Hello, World!</h1>
    <h2>Ivan Dvoretskyi</h2>
    <h3>Course: Разом зі спеціальністю, Group: УБД-21</h3>
    <script src="script.js"></script>
</body>
</html>
```

### Розбір HTML елементів:

#### 1. **DOCTYPE та HTML структура**
```html
<!DOCTYPE html>
<html>
```
**Призначення:** 
- `<!DOCTYPE html>` - оголошення стандарту HTML5
- `<html>` - кореневий елемент документа

#### 2. **HEAD секція**
```html
<head>
    <title>My First Lab Website</title>
    <link rel="stylesheet" href="style.css">
</head>
```
**Компоненти:**
- **`<title>`** - заголовок вкладки браузера
- **`<link rel="stylesheet">`** - підключення зовнішнього CSS файлу
- **`href="style.css"`** - відносний шлях до файлу стилів

#### 3. **BODY секція**
```html
<body>
    <h1>Hello, World!</h1>
    <h2>Ivan Dvoretskyi</h2>
    <h3>Course: Разом зі спеціальністю, Group: УБД-21</h3>
    <script src="script.js"></script>
</body>
```
**Елементи контенту:**
- **`<h1>`** - головний заголовок (найвищий рівень)
- **`<h2>`** - підзаголовок другого рівня (ім'я студента)
- **`<h3>`** - підзаголовок третього рівня (інформація про курс)
- **`<script src="script.js">`** - підключення JavaScript файлу

### Семантична структура:
**Ієрархія заголовків:**
- H1: Привітання (Hello, World!)
- H2: Особиста інформація (ім'я)
- H3: Академічна інформація (курс і група)

## Аналіз CSS стилів (style.css)

```css
body {
    font-family: Arial, Helvetica, sans-serif;
    background-color: white;
    margin: 0;
    padding: 0;
    display: flex;
    flex-direction: column;
    justify-content: center;
    align-items: center;
    height: 100vh;
}
```

### Стилізація BODY:

#### 1. **Типографіка**
```css
font-family: Arial, Helvetica, sans-serif;
```
**Призначення:** Встановлення шрифту з fallback опціями
- **Arial** - основний шрифт
- **Helvetica** - альтернатива для macOS
- **sans-serif** - загальна категорія без засічок

#### 2. **Основні стилі**
```css
background-color: white;
margin: 0;
padding: 0;
```
**Функції:**
- **`background-color: white`** - білий фон сторінки
- **`margin: 0; padding: 0`** - скидання браузерних відступів за замовчуванням

#### 3. **Flexbox layout**
```css
display: flex;
flex-direction: column;
justify-content: center;
align-items: center;
height: 100vh;
```
**Flexbox властивості:**
- **`display: flex`** - включення flex-контейнера
- **`flex-direction: column`** - вертикальне розташування елементів
- **`justify-content: center`** - центрування по головній осі (вертикально)
- **`align-items: center`** - центрування по поперечній осі (горизонтально)
- **`height: 100vh`** - висота на весь viewport (100% висоти екрану)

### Стилізація заголовків:

#### H1 стилі:
```css
h1 {
    color: blue;
    font-size: 2rem;
    text-align: center;
}
```
**Властивості:**
- **`color: blue`** - синій колір тексту
- **`font-size: 2rem`** - розмір шрифту 2rem (відносно кореневого елемента)
- **`text-align: center`** - центрування тексту

#### H2 стилі:
```css
h2 {
    color: green;
    font-size: 1.5rem;
}
```
**Особливості:**
- **`color: green`** - зелений колір
- **`font-size: 1.5rem`** - менший за H1 розмір

#### H3 стилі:
```css
h3 {
    color: red;
    font-size: 1.2rem;
}
```
**Характеристики:**
- **`color: red`** - червоний колір
- **`font-size: 1.2rem`** - найменший розмір серед заголовків

### Колористична схема:
- **H1 (головний):** Синій - привертає увагу
- **H2 (ім'я):** Зелений - позитивність, особистість  
- **H3 (курс):** Червоний - важливість, академічність

## Аналіз JavaScript (script.js)

```javascript
console.log("Welcome to the First Lab Website!");
```

### Призначення та функціонал:

#### 1. **Console logging**
**Функція:** Виведення повідомлення в консоль браузера
**Використання:** 
- Налагодження коду
- Підтвердження завантаження скрипта
- Відстеження виконання програми

#### 2. **Розташування скрипта**
**Важливо:** Скрипт розташований в кінці `<body>`, що забезпечує:
- Завантаження HTML перед виконанням JavaScript
- Доступність DOM елементів для маніпуляцій
- Кращу продуктивність завантаження сторінки

## Технічні концепції

### 1. **Веб-стандарти**

#### HTML5 DOCTYPE:
```html
<!DOCTYPE html>
```
**Значення:**
- Сучасний стандарт HTML
- Забезпечення коректного відображення у браузерах
- Активація стандартного режиму рендеринга

#### Семантичні елементи:
- **Заголовки (h1-h6):** Структурування контенту
- **Ієрархічність:** Логічна організація інформації

### 2. **CSS методології**

#### Reset стилів:
```css
margin: 0;
padding: 0;
```
**Мета:** Уніфікація відображення в різних браузерах

#### Responsive units:
```css
font-size: 2rem;    /* Відносно кореневого елемента */
height: 100vh;      /* Відносно висоти viewport */
```
**Переваги:**
- Адаптивність до різних розмірів екранів
- Масштабованість інтерфейсу

#### Modern CSS Layout (Flexbox):
```css
display: flex;
flex-direction: column;
justify-content: center;
align-items: center;
```
**Можливості:**
- Сучасне позиціонування елементів
- Легке центрування контенту
- Гнучка адаптація до розмірів контейнера

### 3. **JavaScript інтеграція**

#### Зовнішні файли:
```html
<script src="script.js"></script>
```
**Переваги:**
- Розділення коду та структури
- Можливість кешування браузером
- Повторне використання в інших проектах

#### Console API:
```javascript
console.log("Welcome to the First Lab Website!");
```
**Застосування:**
- Налагодження веб-додатків
- Логування подій та помилок
- Моніторинг виконання коду

## Принципи веб-розробки

### 1. **Progressive Enhancement (Прогресивне покращення)**
**Базовий рівень:** HTML контент доступний завжди
**Покращення:** CSS та JavaScript додають функціонал
**Результат:** Сайт працює навіть при відключених стилях/скриптах

### 2. **Accessibility (Доступність)**
**Семантична розмітка:** Правильне використання заголовків
**Структурованість:** Логічна ієрархія контенту
**Читабельність:** Контрастні кольори та розбірливі шрифти

### 3. **Performance (Продуктивність)**
**Мінімалізм:** Простий код без зайвих елементів
**Оптимізація:** Зовнішні файли для кешування
**Швидкість:** Швидке завантаження та відображення

## Ключові концепції для захисту

### 1. **HTML питання**
**П:** Що означає DOCTYPE html?
**В:** Оголошення стандарту HTML5, яке вказує браузеру використовувати сучасний режим рендеринга

**П:** Чому важливий порядок елементів в HEAD?
**В:** Деякі елементи (charset, viewport) мають виконуватись першими для коректної обробки документа

**П:** Навіщо підключати CSS у HEAD, а JavaScript перед закриттям BODY?
**В:** CSS блокує рендеринг, тому завантажується спочатку. JS може маніпулювати DOM, тому виконується після завантаження HTML

### 2. **CSS питання**
**П:** Що таке Flexbox і як він працює?
**В:** Сучасна система компонування, що дозволяє легко розподіляти та вирівнювати елементи в контейнері по осях

**П:** Чим відрізняється rem від px?
**В:** rem - відносна одиниця до розміру кореневого елемента, px - абсолютна. rem забезпечує кращу масштабованість

**П:** Що робить justify-content та align-items у Flexbox?
**В:** justify-content вирівнює по головній осі, align-items - по поперечній осі

### 3. **JavaScript питання**
**П:** Що робить console.log?
**В:** Виводить повідомлення у консоль розробника браузера для налагодження та відстеження виконання коду

**П:** Чому скрипт розташований в кінці body?
**В:** Забезпечує завантаження DOM перед виконанням JavaScript, що дозволяє маніпулювати елементами сторінки

### 4. **Загальні концепції**
**П:** Що таке separation of concerns?
**В:** Принцип розділення різних аспектів програми: HTML (структура), CSS (стилі), JS (поведінка)

**П:** Як забезпечити кросбраузерність?
**В:** Використання стандартних технологій, reset стилів, тестування в різних браузерах
